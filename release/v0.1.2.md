## loglm v0.1.2

This release focuses on prompt-agent operability, visibility, and project governance for real-world use.

### Highlights

- Added prompt-agent version metadata support
  - Reads version from:
    - `<!-- prompt-agent-version: 1.2.3 -->`
    - or YAML front matter `prompt_agent_version: 1.2.3`
- Added `loglm agent list --verbose`
  - Shows installed prompt file and `prompt_agent_version`
- Improved readability of verbose output
  - Multi-line block format per agent entry
- Added managed escalation-first policy block to generated project prompt files
  - On permission/sandbox failures, request escalation first before switching to alternatives
- Added explicit project metadata files
  - `LICENSE` (`GPL-3.0-or-later`)
  - `AUTHORS`

### Notes

- Prompt-agent version keys are shared/common keys:
  - `prompt-agent-version` (HTML comment)
  - `prompt_agent_version` (YAML front matter)
- Agent-specific keys like `gamer-pat-version` are not parsed by `loglm`.

### Platform Status

- Tested:
  - macOS
  - Ubuntu
  - Ubuntu on Lima (macOS)
  - Ubuntu on WSL2
- Untested:
  - Raspberry Pi OS
  - Chrome OS (Linux container / Crostini)
