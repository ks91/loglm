## loglm v0.1.1

Patch release focused on developer workflow and prompt-agent authoring/testing.

### Highlights

- Added `--version` / `-v` to `loglm`
  - `loglm --version` now reports the installed version.
- Added `scripts/dev-install.sh` for branch-based development install
  - Uses current branch by default
  - Supports `--branch`, `--repo`, and `--print-only`
- Added local repository source support for prompt-agent install
  - `loglm agent install <local_repo_path>`
  - Useful for private/internal prompt-agent development and testing
- Improved managed block guidance for prompt-agent execution
  - Clarifies that agents must use local installed `<REPO>.md` in current directory
  - Explicit recovery guidance when the installed prompt file is missing

### Platform Status

- Tested:
  - macOS
  - Ubuntu
  - Ubuntu on Lima (macOS)
  - Ubuntu on WSL2
- Untested:
  - Raspberry Pi OS
  - Chrome OS (Linux container / Crostini)

### Notes

- Claude Code: after Homebrew/npm install, native install is recommended.  
  Run `claude install` to migrate to native installation.
- Gemini resume behavior can vary by environment; if context is not restored, run `/resume` or `/chat resume` inside Gemini.

### Install / Uninstall

```bash
curl -fsSL https://raw.githubusercontent.com/ks91/loglm/main/install.sh | bash
curl -fsSL https://raw.githubusercontent.com/ks91/loglm/main/uninstall.sh | bash
```
